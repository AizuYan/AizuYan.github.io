PHP7已经出来很久了，性能也有了大幅度提升。

性能的提升就意味着更快的响应，更少的机器资源投入。

甚至在某些场景PHP7的性能已经超过了facebook的HHVM。

这还得感谢开源社区那一票牛逼人物，包括PHP7的核心开发者鸟哥。

下图是摘自鸟哥博客 PHP7相对于PHP5的性能比对：

项目现状

目前我所在的项目，使用的PHP版本是v5.4，真的已经是PHP的爷爷辈儿了，看项目中的有些代码都是11年写的，代码也是祖传的了，除了性能之外，还会有些问题，比如想引入自动化测试，使用PHPUnit，你会发现，开源社区维护的版本中已经不再继续支持v5.4了，是不是有种被抛弃的感觉，其实这只是主管感受罢了，是我们抛弃了v7.x，是我们抛弃了PHPunit。这只是个例子，等祖传代码出现问题的时候，你会发现开源社区中已经找不到持续维护中的类库版本了，真特么尴尬。因此，决定将项目的PHP版本升级至v7.x。

存在的问题

PHP虽然说版本向下兼容，比如PHP5的代码在PHP7中也是可以运行的，但是这只是大多数代码，版本升级之后可能有些变动，比如废弃哪个函数，移除哪个函数等，比如mysql_*()系列函数在PHP5.5中废弃了但还是可以用，PHP7中就移除了，完全用不了了，类似的还有很多。详细迁移注意事项可以这里从 PHP 5.4.x 迁移到 PHP 5.5.x，从PHP5.4.x到PHP7.x.x有三次迁移变动，注意看图片右侧一栏中白框框起来的内容

所以，迁移的时候，不只是将PHP版本从v5.x升级到v7.x，如果你们用的都是开源社区的类库，升级对应的类库，剩下的应该没几个问题了(但是跟主流版本差的比较远升级开源类库可能比较麻烦)，但是如果是自己写的类库很多，而且都是祖传代码，可能就问题比较多了，需要将几次升级前一中的注意事项全部看一遍。

老项目，代码数量庞大，如果一个个去看，想必不是一件轻松地工作，可能会出人命，累死人，还保证不了效果，可能会有疏漏。

绝处逢生

查了很多资料，问了好多前辈，终于在PHP内核交流群中有个叫@白冰的朋友推荐了PHP-Migration这个开源项目，静态分析扫描代码，可以将升级之后的潜在问题都扫描出来，作者的原话是做到严谨不遗漏，并做到尽可能精准，真是绝处逢生啊，天台的位置可以让给足彩赌博的哥们了，哈哈哈。

PHP-Migration 原理

PHP-Migration的核心原理，就是将待校验的代码进行词法分析、语法分析、生成AST(Abstract Syntax Tree)树，然后对AST进行静态分析，校验升级过程中的每一个不兼容变动，然后给与对应的提示。

设计原理

PHP-Migration设计的比较巧妙，每次版本迁移都是独立的，但之间又有依赖关系，比如要校验PHP70的规则，他会校验PHP70相关信息并依赖PHP56相关信息，校验PHP56相关信息并依赖PHP55的相关信息，然后递归解析…这一一层层下去，完全可以满足各个版本的PHP迁移需求。具体依赖关系如下所示，v*表示的是对应版本的变动、升级迁移后注意的信息，里面维护的都是类似我上面例子中的MyTester，里面是对应的校验规则，to*表示的是校验到那个版本。

to55 v55,to54
